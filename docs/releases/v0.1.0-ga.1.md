# v0.1.0-ga.1 Release Notes

发布日期：2026-06-13
状态：GA patch
基线版本：v0.1.0-ga

## Release Verdict

`v0.1.0-ga.1` 是非 OCR 加密/代码保护 GA 的发布证据闭环补丁版本。

它不扩大 `v0.1.0-ga` 的产品能力边界，不新增 OCR、QR、cross-media、remote-KMS 服务端或第三方逆向评估结论。它解决的是：`v0.1.0-ga` 已具备本地 GA 证据后，进一步把 GitHub Actions、GitHub Release artifact、下载后 replay 复验和文档归档串成可审计的发布事实。

## What Changed Since v0.1.0-ga

```text
新增 GitHub Actions 级 non-ocr-ga-landing 发布门禁。
新增 landing gate replay，支持下载 GitHub artifact 后重新验证 smoke report 和 promotion_artifact_bundle.zip。
增强 promotion_artifact_bundle.zip 审计：manifest generated_at_utc、entry source_path、必需条目、未声明 zip 条目、sha256、release_bundle layout_version 均 fail-closed。
landing gate report 新增 artifact_manifest.sha256 清单和 json_schema_version_audit_passed 摘要字段。
GitHub workflow artifact 明确保留 90 days。
GitHub Release v0.1.0-ga 已挂载三份关键证据附件。
发布 runbook 补齐 artifact 路径、保留周期、下载后复验和停止条件。
已完成 landing/trust hardening 施工计划归档，下一阶段只保留 post-GA trust hardening checklist 作为 working 候选池。
```

## What This Solves

相对于 `v0.1.0-ga`，本版本解决了四个发布治理问题：

```text
1. 从“本地证据通过”推进到“GitHub Actions artifact 可下载、可复验”。
2. 从“bundle 存在”推进到“bundle 每个条目可按 manifest 和 sha256 审计”。
3. 从“Release 说明存在”推进到“GitHub Release 附件存在并可作为发布事实”。
4. 从“working plan 已实施”推进到“已实施计划归档，下一阶段 working 边界清楚”。
```
